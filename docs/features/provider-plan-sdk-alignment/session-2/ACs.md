## Design summary

Two layers, one ephemeral and one durable:

| | Lifespan | Renders |
|---|---|---|
| **PaymentConfirmModal** | Ephemeral — alive only during an active payment attempt on the current tab | `preparing` (server building Stripe session) → `polling` (Stripe tab open, waiting on plan→ACTIVE) → optional `error` (anything went wrong) |
| **PendingCard** | Durable — reflects whatever `plan.state.status === "PENDING"` represents | Two substates differentiated only by `state.statusInfo.descText` ("Confirming your payment…" or "Setting up your store…") — same shape, same actions, same spinner |

A single click on a subscribe/convert action with a `modalSlug` resolving to a `paymentConfirm` modal:

1. Synchronously: `window.open("about:blank", "_blank")` (preserves popup-blocker permission) + mount the modal in `preparing` state.
2. Asynchronously: `fetch(action.endpoint)` — platform sets `plan.state` to PENDING + creates Stripe session + returns `{ stripeUrl }`.
3. On response: `stripeTab.location.href = stripeUrl` + transition modal to `polling`. The modal then polls (`router.refresh()`) and watches `plan.state.status`.
4. ACTIVE → modal unmounts. NONE (failure/cancel/tab-closed) → modal flips to `error` with generic copy and Try Again + Close CTAs. Try Again re-runs step 1.

No page freeze. The PendingCard behind the modal is just a regular card.
